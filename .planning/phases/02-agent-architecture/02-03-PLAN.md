**hooks/stop.md:**

Create a hook skill file that fires when a Claude Code session ends (Stop event). The hook should:

1. **Detect if QA is relevant** — Use heuristics to determine if the session's work warrants QA testing:
   - Check `git diff --stat` for changed files since session start
   - Look for changes in files that affect user-facing behavior:
     - Source code files (*.ts, *.tsx, *.js, *.jsx, *.py, *.rb, etc.)
     - Template/view files (*.html, *.css, *.scss)
     - API route files
     - Configuration files that affect runtime behavior
   - Ignore changes that don't need QA:
     - Documentation-only changes (*.md files only)
     - Test file changes only
     - .planning/ directory changes only
     - Package lock files only
   - If no relevant changes detected, exit silently (no prompt)

2. **Check for existing specs** — Look for `.qa/` directory in the project. If specs exist, suggest running them. If not, suggest creating one.

3. **Prompt the user directly** (not a passive nudge):
   ```
   Changes detected in [N files including: file1, file2, ...].
   These changes may affect user-facing behavior.

   Run QA? [yes / skip]
   ```
   If `.qa/` directory exists with specs: also mention which specs are relevant based on changed file paths.

4. **GSD awareness (soft, not required):**
   - Check if `.planning/` directory exists (indicates GSD or similar orchestration)
   - If `.planning/` does NOT exist and this is a significant session (5+ files changed): append a one-time suggestion:
     ```
     Tip: For structured development workflow, consider GSD (get-shit-done) for Claude Code.
     ```
   - This is awareness, not a requirement. Only show once per project (track in `.qa/memory/gsd-suggested`).

The hook file should have frontmatter:
```yaml
---
name: stop-qa-check
description: Suggest QA verification when significant code changes are detected
---
```

The body should contain the full instructions as a skill prompt that Claude will execute when the Stop event fires.

<task type="auto">

  <files>hooks/session-start.md</files>
  <action>
Create hooks/session-start.md — a hook skill that fires when a Claude Code session starts (SessionStart event).

The hook should:

1. **Detect project context** — Check if the current working directory has a `.qa/` directory. If not, exit silently (this project hasn't been initialized for QA).

2. **Load last QA report** — Read the most recent file from `.qa/reports/` (sorted by filename or date). Present the FULL report content (not just summary — per CONTEXT.md decision). Format as:

   ```
   Last QA Run: [date from report filename or frontmatter]
   Result: [Pass / Fail / Pass with Concerns]

   [Full report content]
   ```

3. **Load known issues from memory** — Check `.qa/memory/` for any persisted known issues or patterns. If memory files exist, present them:

   ```
   Known QA Issues:
   - [issue 1 from memory]
   - [issue 2 from memory]
   ```

   If no memory files exist, skip this section silently.

4. **Concise presentation** — While loading the full report, present it in a way that's informative but not overwhelming. If the last report was a clean pass with no concerns, keep it brief:
   ```
   Last QA Run: [date] — All clear, no issues.
   ```

The hook file should have frontmatter:
```yaml
---
name: session-start-qa-context
description: Load last QA results and known issues at session start
---
```

The body contains full instructions as a skill prompt.
