**What to remember (agent's discretion):**
- Project-specific patterns (e.g., "this app uses React, Next.js API routes, Prisma")
- Known issues and quirks (e.g., "login page has a 2s delay on first load — known, not a bug")
- Test history patterns (e.g., "checkout flow has been flaky for 3 sessions")
- Auto-expire stale entries after 30 days of no update

**Where QA data lives:**
- Project-local: `{project}/.qa/reports/` — test reports (already established)
- Project-local: `{project}/.qa/memory/` — project-specific QA memory files
- Memory file format: markdown with YAML frontmatter (date, type, project)

**Memory namespace isolation:**
- All QA memory files MUST be within `.qa/` directories
- Never write to `~/.claude/memory/` or any path outside `.qa/`
- This prevents interference with user's own memory, other plugins, etc.

Add a new section "## Background Execution" after the Memory section:

**Default mode:** Inline (foreground) — user watches tests run in real time.

**Background mode:** When invoked with `--background` flag:
- Run tests silently, write results to `{project}/.qa/reports/`
- On failure: notify the user with a summary of what failed
- On all-pass: write report silently, no interruption
- Concurrency: run one background test session at a time (avoid resource contention with Playwright)

Update the "## Session Structure" section to add step 5:
5. Update QA memory with any new patterns, issues, or observations worth persisting
