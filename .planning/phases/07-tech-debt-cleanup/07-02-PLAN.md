<task type="auto">
  <name>Task 1: Fix Stop hook spec path and skill syntax</name>
  <files>hooks/stop.md</files>
  <action>
Two bugs to fix in hooks/stop.md:

**Bug 1 — Wrong spec path (line 56-57):**

Current (WRONG):
```bash
ls .qa/specs/ 2>/dev/null
```

The project stores specs directly in `.qa/` (not `.qa/specs/`). This check will always find nothing, making the hook think no specs exist. Fix to:
```bash
ls .qa/*.md 2>/dev/null
```

This correctly checks for markdown spec files in the `.qa/` directory. Also update the surrounding text — line 55 says "check which spec files exist" which is fine, but the comment about checking `.qa/specs/` should reference `.qa/*.md` instead.

**Bug 2 — Wrong skill syntax (line 79):**

Current (WRONG):
```
- If **yes** → remind user to run: `/qa-run` (or describe the QA skill to run)
```

The correct skill syntax uses a colon, not a hyphen. Fix to:
```
- If **yes** → remind user to run: `/qa:run` (or `/qa:run --project .` if specs exist)
```

These two bugs together break the entire GSD integration flow: the hook can't find specs (wrong path) and suggests the wrong command (wrong syntax).
  </action>
  <verify>
Run: grep "qa/specs/" hooks/stop.md (should return NO matches); grep "/qa-run" hooks/stop.md (should return NO matches); grep "/qa:run" hooks/stop.md (should return a match); grep ".qa/\*.md" hooks/stop.md (should return a match)
  </verify>
  <done>Stop hook checks `.qa/*.md` for specs and suggests `/qa:run` with correct colon syntax</done>
</task>
