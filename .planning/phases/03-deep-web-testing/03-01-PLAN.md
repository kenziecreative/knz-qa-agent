  <files>agents/qa-tester.md</files>
  <action>
Add a new "## Network Intelligence" section to agents/qa-tester.md, placed after the existing "### 2. During Each Interaction" subsection (which already mentions inspecting network requests — this section deepens that).

The section should cover:

**Network Request Monitoring (observe by default):**
- During every interaction, inspect network requests via Playwright's network monitoring
- Check response status codes — flag 4xx/5xx responses that aren't expected
- Track response times — flag slow calls using sensible defaults by request type:
  - API/XHR calls: > 1s is slow, > 3s is very slow
  - Page navigations: > 2s is slow, > 5s is very slow
  - Static assets: > 500ms is slow
  - These are defaults; specs can override thresholds
- Correlate failed requests with UI behavior — if a network request fails, check what the UI shows (error message? loading spinner stuck? silent failure with no user feedback?)
- Detect over-fetching — if an API response contains significantly more data than the UI renders (e.g., returns 50 fields but UI shows 5), note it as a possible optimization opportunity (Low confidence)
- Check for mixed content — flag any HTTP resources loaded on HTTPS pages

**Tiered approach (consistent with Phase 2 patterns):**
- Default: Observe network traffic passively during normal test flows
- Active simulation: Only intercept/block routes when the spec explicitly includes error recovery scenarios (defer to Plan 04 for the error simulation methodology)

**Reporting network findings:**
- Network issues should be reported with the same confidence framework (certainty x severity)
- Failed requests correlated with broken UI = High confidence
- Slow requests with no visible impact = Low confidence
- Over-fetching = Low confidence (optimization, not a bug)
- Mixed content = Medium confidence (security concern)

Also update the existing "Security Observations" section: remove the bullet about "Mixed content warnings" since it's now covered more thoroughly in Network Intelligence. Remove the bullet about "API responses containing more data than the UI needs" since that's now over-fetching detection in Network Intelligence.
